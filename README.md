# Working on making a CLAUD.md file which causes Claude to look at and create code the way that I do

The process has been, so far, as follows:

- Start with a working moral, and philosophical framework. Meant for humans as much as for agents (PHILOSOPHY.md)
	- Began with an incomplete list of things I consider to be important when writing code.
	- Fed that to Claude, explained what I was doing
	- Worked in circles getting close to something that I could work with as a starting point
	- Introduced some documents I have written in the past on writing code
	- Continued to refine until I had something that eyeballed as workable, but was pretty human centric
- Attempt to remove the human oriented verbiage from the framework into an uncompressed CLAUDE.md file
	- Pretty straight forward. I asked Claude if what would become PHILOSOPHY.md wasn't a bit long for a CLAUDE.md file
	- Had it remove the human elements.
	- Had it go back over and double check nothing important was left out (it was, and some might still be)
	- Had it shore things up
- Compress, or distill, the resulting files
	- Had it use a bunch of sub agents to hold several tournaments
	- Had the main chat judge the output by the full expanded PHILOSOPHY.md documents standards
	- Had it go for a few rounds until we found something that seemed to both be good and small
	- Candidates recorded here as COMPRESSED-test-X.md

Was this a good way to do this? Honestly I don't know. This is just an experiment right now.
